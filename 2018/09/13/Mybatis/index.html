Mybatis

支持普通SQL查询，存储过程和高级映射关系的优秀持久层框架

核心配置文件mybatisconfig.xml

注册映射文件EmpMapper.xml

常见的四个标签：select，update，insert，delete

每个标签中都有 id 属性，在同一个 mapper 文件中 id 不允许重复；

参数 parameterMap 已经被废弃，现在其存在的目的就是为了兼容前期的项目；

参数 parameterType 支持很多的类型，例如 int、Integer、String、Double、List、Map 或者实体对象等；

返回值 resultType 用于简单的类型；

返回值 resultMap 用于复杂的类型；

当参数和返回值是集合的时候，其声明的是集合中的元素类型；

SQL 语句不区分大小写，它默认使用 PrepareStatement，预编译，可以防止 SQL 注入。

调用方式

1.获取factory

2.SqlSession实例化dao层的实现类对象

Sql标签

当多出使用相同的字段时，使用SQL标签完成底层的字符串拼接


id,name,password

select from person

映射文件

1.设置namespace命名空间，目的是为了区分映射文件中的方法
2.结果集resultMap是MyBatis最大的特色，对象的ORM就由其来转换：
结果集中包括主键id和普通属性result

结果集中，常用的两个属性分别为：property，表示实体的属性；column，表示 SQL 查询的结果集的列。
